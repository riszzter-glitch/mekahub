Pengenalan PLC & Sistem Otomasi

Sejarah, definisi, komponen dasar, dan aplikasi PLC di industri modern — fondasi wajib sebelum memprogram.

Kelas: XI/XII TEI & TPM
Durasi: 2 × 45 menit
Level: Dasar
Topik: 6 Sub-Topik

1. Tujuan Pembelajaran

  1. Menjelaskan pengertian dan fungsi PLC dalam sistem otomasi industri.
  2. Menelusuri sejarah perkembangan PLC dari relay elektromekanis hingga era digital.
  3. Mengidentifikasi komponen utama PLC beserta fungsinya masing-masing.
  4. Membandingkan sistem kontrol relay konvensional dengan sistem berbasis PLC.
  5. Menyebutkan minimal 5 aplikasi nyata PLC di industri manufaktur.
  6. Memahami siklus kerja (scan cycle) PLC secara umum.

2. Apa itu Sistem Otomasi?

Sistem otomasi adalah penggunaan teknologi — komputer, perangkat elektronik, dan robotika — untuk mengendalikan mesin dan proses industri dengan sedikit atau tanpa campur tangan manusia. Tujuannya adalah meningkatkan efisiensi, keselamatan, akurasi, dan konsistensi produksi.

💡 Tingkatan Otomasi Industri Otomasi terbagi menjadi tiga level: (1) Fixed Automation – mesin khusus untuk satu tugas tetap; (2) Programmable Automation – dapat dikonfigurasi ulang, contoh: PLC; (3) Flexible Automation – sistem robot canggih yang dapat beradaptasi otomatis.

3. Pengertian PLC

PLC (Programmable Logic Controller) adalah komputer industri yang dirancang khusus untuk menggantikan sistem kontrol relay konvensional. PLC mampu membaca sinyal input dari sensor, memproses logika sesuai program yang telah dibuat, lalu mengeksekusi perintah ke aktuator/output.

PLC memiliki keunggulan utama: dapat diprogram ulang kapan saja tanpa mengubah wiring fisik, sehingga perubahan logika kontrol menjadi sangat cepat dan murah. Ini berbeda dengan panel relay konvensional yang harus di-rewire manual setiap kali logika berubah.

4. Sejarah Singkat PLC

Tahun Tonggak Sejarah
1968 Dick Morley (Modicon) menciptakan PLC pertama: Modicon 084, pesanan General Motors untuk menggantikan ribuan relay di lini produksi otomotif.
1970-an PLC mulai diproduksi massal oleh Allen-Bradley dan Siemens. Masih berbasis prosesor 8-bit dengan memory sangat terbatas.
1980-an Standarisasi bahasa pemrograman IEC 61131-3 mulai dirancang. Muncul Ladder Diagram, Function Block Diagram, dan Structured Text.
1990-an PLC mulai terhubung ke jaringan industri (fieldbus: Profibus, DeviceNet). Integrasi dengan SCADA/HMI.
2000-an PLC generasi baru: lebih kecil, lebih cepat, mendukung Ethernet industrial (PROFINET, EtherNet/IP). Siemens S7-1200/1500 lahir.
2020-an Era Industrial IoT: PLC terhubung cloud, integrasi dengan AI/ML untuk predictive maintenance dan smart factory (Industri 4.0).

5. Komponen Utama PLC

🧠 CPU (Central Processing Unit)

Otak PLC. Mengeksekusi program logika secara berulang (scan cycle), mengelola memori, dan berkomunikasi dengan modul I/O. Kecepatan CPU diukur dalam scan time (ms).

💾 Memori

Menyimpan program logika (OB, FC, FB) dan data variabel (Timer, Counter, flag). Terdiri dari RAM (sementara) dan Flash/EEPROM (permanen, tidak hilang saat mati listrik).

📥 Modul Input (DI/AI)

Menerima sinyal dari sensor. DI (Digital Input): kontak ON/OFF dari tombol, limit switch, proximity. AI (Analog Input): sinyal 0–10V atau 4–20mA dari sensor suhu, tekanan, level, dll.

📤 Modul Output (DO/AO)

Mengirim sinyal ke aktuator. DO (Digital Output): menghidupkan relay, kontaktor, lampu, solenoid. AO (Analog Output): mengatur kecepatan inverter, posisi katup proporsional.

⚡ Power Supply

Mengubah tegangan AC (220V) menjadi DC (24V) untuk memberi daya CPU dan modul I/O. Harus stabil dan dilindungi MCB tersendiri.

🖥️ HMI / Programming Device

Komputer laptop/PC dengan software TIA Portal (Siemens) atau RSLogix (Allen-Bradley) untuk membuat, mengunduh, dan memonitor program PLC.

6. Siklus Kerja (Scan Cycle) PLC

PLC bekerja secara siklik, terus mengulang 4 tahap dalam urutan:

  1. Input Scan – CPU membaca status semua terminal input (sensor, tombol) dan menyimpannya ke image register.
  2. Program Execution – CPU menjalankan program logika (Ladder/FBD) dari baris pertama hingga terakhir berdasarkan data image register.
  3. Output Scan – Hasil logika program ditulis ke output image register, lalu dikirim ke terminal output fisik (aktuator).
  4. Housekeeping – CPU melakukan komunikasi (HMI, jaringan), self-diagnostik, dan memperbarui timer internal.
⚠️ Scan Time & Performa Scan time tipikal PLC modern antara 1–10 ms. Program yang terlalu panjang atau banyak operasi berat dapat memperlambat scan time dan menyebabkan watchdog timeout. Pantau selalu "Cycle Time" di TIA Portal saat commissioning.

7. Perbandingan: Relay Konvensional vs PLC

Aspek Relay Konvensional PLC
Perubahan Logika Harus re-wiring fisik (lama, mahal) Ubah program software (cepat, murah)
Diagnosis Kerusakan Manual, memeriksa satu per satu Diagnostik real-time di layar HMI
Ukuran Panel Besar, banyak komponen Kompak, satu unit PLC menggantikan ratusan relay
Keandalan Kontak mekanis aus, rentang 10–30 juta operasi Solid-state, tanpa bagian bergerak (sangat andal)
Biaya Awal Murah untuk sistem sederhana Lebih mahal, tapi hemat jangka panjang
Kemampuan Timer/Counter Timer/Counter terpisah (biaya ekstra) Ratusan Timer & Counter virtual di dalam CPU

8. Aplikasi PLC di Industri

  • Otomotif: Kontrol robot las, conveyor assembly, pengecatan otomatis di pabrik mobil.
  • Makanan & Minuman: Pengisian botol otomatis, pasteurisasi, pengemasan, dan pelabelan.
  • Minyak & Gas: Kontrol pompa, katup pengaman, sistem ESD (Emergency Shutdown).
  • Tekstil: Kontrol mesin rajut, tenun, pencelupan warna otomatis berbasis resep.
  • Air & Sanitasi: Kontrol pompa air, level tangki, pengolahan air limbah (WWTP).
  • Gedung (BMS): Kontrol lift, HVAC, sistem pemadam kebakaran (Fire Suppression System).
