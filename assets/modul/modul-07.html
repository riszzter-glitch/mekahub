📊 MODUL 07 – MENENGAH

Function Block Diagram (FBD)

Bahasa pemrograman PLC berbasis blok fungsi grafis — lebih intuitif untuk logika kompleks, pengolahan data, dan sistem kontrol hierarkis.

Kelas: XII TEI & TPM
Durasi: 2 × 45 menit
Level: Menengah
Topik: 6 Sub-Topik

1. Tujuan Pembelajaran

  1. Menjelaskan konsep dan keunggulan Function Block Diagram dibandingkan Ladder Diagram.
  2. Mengidentifikasi blok fungsi dasar FBD: AND, OR, NOT, SR, RS, TON, CTU.
  3. Merancang program FBD sederhana untuk logika Boolean dan kontrol ON/OFF.
  4. Memahami konsep Function Block (FB), Function (FC), dan Organization Block (OB) dalam struktur program PLC.
  5. Membuat Function Block reusable yang dapat digunakan berulang kali di program lain.
  6. Membandingkan kapan sebaiknya menggunakan FBD vs Ladder Diagram dalam proyek nyata.

2. Lima Bahasa Pemrograman PLC (IEC 61131-3)

LD – Ladder Diagram

Grafis, berbasis simbol relay. Ideal untuk logika Boolean dan kontrol digital sederhana. Paling populer di industri.

FBD – Function Block Diagram

Grafis, berbasis blok yang dihubungkan. Ideal untuk alur data, pengolahan sinyal analog, dan kontrol PID.

SFC – Sequential Function Chart

Grafis, berbasis diagram alir. Sangat cocok untuk proses sekuensial (langkah demi langkah).

ST – Structured Text

Teks, seperti bahasa Pascal/C. Paling powerful untuk kalkulasi matematis, array, dan algoritma kompleks.

IL – Instruction List

Teks, seperti bahasa Assembly. Level rendah, jarang digunakan di era modern. Digantikan ST.

Kombines

TIA Portal mendukung penggunaan semua bahasa dalam satu project. Pilih bahasa yang paling sesuai per network atau blok.

3. Apa itu Function Block Diagram (FBD)?

FBD (Function Block Diagram) adalah bahasa pemrograman PLC yang menggunakan blok-blok grafis (kotak dengan pin input dan output) yang dihubungkan satu sama lain menggunakan garis (wire/connection). Setiap blok mewakili fungsi tertentu — logika, matematika, kontrol, atau komunikasi.

FBD sangat mirip dengan diagram rangkaian elektronika digital atau blok diagram kontrol. Aliran sinyal berjalan dari kiri ke kanan. FBD lebih disukai untuk logika yang melibatkan banyak perhitungan dan pengolahan data dibandingkan Ladder Diagram.

4. Blok Fungsi Dasar FBD

Blok AND – Output TRUE hanya jika SEMUA input TRUE: A ─┐ AND ── Q_AND B ─┘ Blok OR – Output TRUE jika SALAH SATU input TRUE: A ─┐ OR ── Q_OR B ─┘ Blok NOT – Membalik logika: A ── NOT ── Q_NOT Blok SR (Set Dominant) – Set-Reset Latch: BTN_START ─── S ┐ SR ── Q1 ── KM_MOTOR BTN_STOP ─── R1 ┘ Blok TON (Timer On Delay): IN ──── IN ┐ TON ── Q ── (output) T#5s ── PT ┘ ET ──── MW0 (elapsed time)

5. Jenis Blok Program dalam Struktur PLC

OB – Organization Block

Blok program utama yang dipanggil oleh sistem operasi PLC. OB1 = Main program (dipanggil setiap scan cycle). OB35 = Cyclic interrupt (setiap X ms). OB40 = Hardware interrupt (dipicu input).

FB – Function Block

Blok dengan "memory" (instance data block). Nilai timer, counter, dan variabel internal tersimpan antar scan. Ideal untuk kontrol motor reusable: buat sekali → panggil berkali-kali.

FC – Function

Blok tanpa memory permanen. Setiap kali dipanggil, semua variabel lokal di-reset. Ideal untuk fungsi perhitungan murni: konversi engineering unit, kalkulasi PID manual.

DB – Data Block

Blok penyimpanan data murni — tidak berisi logika. Instance DB dibuat otomatis untuk setiap FB. Global DB digunakan untuk berbagi data antar blok program (resep, parameter).

6. Contoh FBD: Kontrol Motor dengan Function Block Reusable

Kelebihan FBD: kita bisa membuat FB_MOTOR_CONTROL sekali, lalu memanggilnya untuk setiap motor dengan parameter yang berbeda-beda.

FB_MOTOR_CONTROL (Function Block – dipanggil untuk setiap motor): Input pins: bStart → BTN_START (BOOL) bStop → BTN_STOP (BOOL) bEstop → E_STOP (BOOL) bOverload → OL_RELAY (BOOL) Output pins: bMotorFwd → KM_FORWARD (BOOL) bFaultLamp → FAULT_LED (BOOL) Internal (dalam Instance DB): bHold : BOOL ← self-holding bit tDelay : TON ← delay timer instance ────────────────────────────── Pemanggilan di OB1 (FBD): Untuk Motor Konveyor 1: [FB_MOTOR_CONTROL] → Instance "DB_Conv1" bStart ← I0.0 bStop ← I0.1 bEstop ← I0.4 bOverload ← I0.6 → bMotorFwd → Q0.0 → bFaultLamp → Q0.3 Untuk Motor Konveyor 2 (copy paste, ganti parameter): [FB_MOTOR_CONTROL] → Instance "DB_Conv2" bStart ← I1.0 bStop ← I1.1 bEstop ← I0.4 (shared E-Stop) bOverload ← I1.6 → bMotorFwd → Q1.0

7. FBD vs. Ladder Diagram – Kapan Menggunakan Mana?

Kriteria Gunakan Ladder Diagram Gunakan FBD
Jenis Logika Logika binary sederhana (ON/OFF) Logika kompleks, multi-input/output
Pengolahan Analog Kurang intuitif Lebih cocok (aliran data terlihat jelas)
Pembaca Program Teknisi elektro/maintenance Engineer kontrol & programmer
Struktur Modular Kurang intuitif untuk FB kompleks Sangat baik untuk reusable block
Visualisasi Mirip diagram relay (familiar) Mirip dataflow diagram (modern)
Standar Industri Paling umum di lapangan Umum di perusahaan engineering & OEM
💡 Tip Profesional Di proyek industri nyata, engineer berpengalaman biasanya mengkombinasikan kedua bahasa: Ladder Diagram untuk network-network digital I/O (interlock, E-Stop, alarm), dan FBD untuk blok kontrol motor reusable, kalkulasi PID, dan pengolahan sinyal analog. Kemampuan menggunakan keduanya adalah nilai tambah yang tinggi.
