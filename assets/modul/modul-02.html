Dasar Pemrograman Ladder Diagram

Simbol-simbol dasar LD, logika Boolean pada PLC, dan cara membuat program pertamamu di TIA Portal.

Kelas: XI/XII TEI & TPM
Durasi: 2 × 45 menit
Level: Dasar
Topik: 8 Sub-Topik

1. Tujuan Pembelajaran

  1. Mengidentifikasi dan menjelaskan simbol-simbol dasar Ladder Diagram (LD).
  2. Memahami hubungan antara logika Boolean (AND, OR, NOT) dan simbol Ladder.
  3. Membuat program Ladder Diagram sederhana untuk kontrol ON/OFF aktuator.
  4. Mengimplementasikan rangkaian self-holding (kunci diri) pada Ladder Diagram.
  5. Membaca dan menginterpretasikan program Ladder Diagram yang sudah ada.
  6. Menjalankan simulasi program pada PLC Simulator (PLCSIM) atau software online.

2. Apa itu Ladder Diagram?

Ladder Diagram (LD) adalah salah satu dari 5 bahasa pemrograman PLC standar IEC 61131-3. Nama "Ladder" (tangga) diambil dari tampilannya yang menyerupai tangga — dua rel vertikal (power rail kanan dan kiri) dihubungkan oleh "anak tangga" (rung) yang masing-masing berisi logika kontrol.

Ladder Diagram merupakan representasi grafis dari sirkuit relay elektromekanis. Setiap rung memiliki kondisi (input/kontak) di sisi kiri dan aksi (output/coil) di sisi kanan. Jika kondisi terpenuhi (arus "mengalir"), maka coil aktif.

💡 Mengapa Ladder Diagram? LD adalah bahasa PLC yang paling populer karena teknisi listrik sudah terbiasa membaca diagram sirkuit relay. Transisi dari relay ke PLC menjadi lebih mudah karena konsepnya hampir identik.

3. Simbol-Simbol Dasar Ladder Diagram

--[ ]--

Kontak NO

Normally Open – aktif (menutup) saat bit bernilai TRUE (1). Default: tidak aktif (terbuka).

--[/]--

Kontak NC

Normally Closed – aktif (menutup) saat bit bernilai FALSE (0). Default: aktif (tertutup).

--( )--

Coil Output

Koil output biasa – bit SET ke TRUE saat rung kondisinya TRUE, RESET ke FALSE saat rung FALSE.

--(S)--

SET Coil

Set (Latch) – bit di-SET menjadi TRUE dan tetap TRUE meski kondisi rung kembali FALSE.

--(R)--

RESET Coil

Reset (Unlatch) – bit di-RESET menjadi FALSE. Digunakan berpasangan dengan SET Coil.

--(N)--

Negative Edge

Aktif hanya SATU SCAN saat terjadi transisi FALSE→TRUE (rising edge) atau TRUE→FALSE (falling edge).

4. Hubungan Logika Boolean & Ladder Diagram

Gerbang Logika Representasi Ladder Keterangan
AND Dua kontak NO disusun seri Output ON hanya jika input A DAN input B sama-sama ON.
OR Dua kontak NO disusun paralel Output ON jika input A ATAU input B (atau keduanya) ON.
NOT Satu kontak NC Output ON jika input TIDAK aktif (inversi logika).
NAND Kontak seri, lalu koil NC NOT dari AND – output OFF hanya jika semua input ON.
NOR Kontak paralel, lalu koil NC NOT dari OR – output OFF jika ada salah satu input ON.
XOR Kombines seri-paralel Output ON jika tepat satu input ON (berbeda satu sama lain).

5. Contoh Program: Kontrol Lampu Dasar

Skenario: Tombol START (I0.0, NO) menyalakan Lampu (Q0.0). Tombol STOP (I0.1, NC) mematikannya.

Network 1 – Kontrol ON/OFF langsung: |--[ I0.0 START ]--[/ I0.1 STOP ]--( Q0.0 LAMP )--| Penjelasan: - I0.0 (NO): Lampu menyala hanya selama tombol START ditekan. - I0.1 (NC): Jika STOP ditekan (TRUE), kontak NC terbuka → Lampu mati. - Tanpa self-holding: saat START dilepas, lampu langsung mati!

6. Rangkaian Self-Holding (Kunci Diri)

Self-holding adalah teknik membuat output "mengunci" dirinya tetap ON meski tombol START sudah dilepas. Ini meniru cara kerja relay dengan kontak auxiliary yang diparalelkan dengan tombol START.

Network 1 – Self-Hold (Seal-in Circuit): |--[ I0.0 START ]--+--[/ I0.1 STOP ]--( Q0.0 LAMP )--| | | |--[ Q0.0 HOLD ]--+ Penjelasan: - Saat START ditekan: Q0.0 = ON - Q0.0 NC menutup dan membuat jalur parallel dengan START - Saat START dilepas: arus tetap mengalir lewat Q0.0 (self-hold) - Saat STOP ditekan: kontak NC I0.1 terbuka → Q0.0 = OFF → self-hold terputus
⚠️ Prioritas STOP Dalam rangkaian self-hold, selalu letakkan kontak STOP (NC) SETELAH jalur paralel. Ini memastikan STOP selalu bisa mematikan output, bahkan jika tombol START masih ditekan bersamaan (prioritas STOP lebih tinggi dari START).

7. Contoh Program Lengkap: 3 Network

Network 1 – START/STOP Motor dengan Self-Hold: |--[ I0.0 START ]--+--[/ I0.1 STOP ]--( Q0.0 MOTOR )--| |--[ Q0.0 HOLD ]--+ Network 2 – Lampu indikator RUN (hijau) saat motor ON: |--[ Q0.0 MOTOR ]--( Q0.1 LAMP_GREEN )--| Network 3 – Lampu indikator FAULT (merah) saat motor OFF: |--[/ Q0.0 MOTOR ]--( Q0.2 LAMP_RED )--|

8. Aturan Penting dalam Ladder Diagram

  • Setiap rung harus memiliki minimal satu output (coil) di ujung kanan.
  • Satu bit output (misal Q0.0) sebaiknya hanya ditulis sekali (satu coil). Output ganda dapat menyebabkan konflik dan hanya nilai terakhir yang berlaku.
  • Kontak dapat digunakan berkali-kali dalam program (NO maupun NC) dari tag yang sama.
  • Urutan rung menentukan prioritas eksekusi — rung atas dieksekusi lebih dahulu.
  • Gunakan komentar rung (Network Title & Comment) untuk mendokumentasikan fungsi setiap rung.
