⏱️ MODUL 04 – DASAR

Timer dan Counter PLC

Penggunaan TON, TOF, TP (Timer) dan CTU, CTD, CTUD (Counter) dalam logika kontrol otomasi industri.

Kelas: XI/XII TEI & TPM
Durasi: 2 × 45 menit
Level: Dasar
Topik: 6 Sub-Topik

1. Tujuan Pembelajaran

  1. Menjelaskan fungsi dan parameter Timer (TON, TOF, TP) dalam program PLC IEC.
  2. Menjelaskan fungsi dan parameter Counter (CTU, CTD, CTUD) dalam program PLC.
  3. Merancang program Ladder Diagram menggunakan Timer untuk penundaan waktu.
  4. Merancang program Ladder Diagram menggunakan Counter untuk menghitung objek.
  5. Mengkombinasikan Timer dan Counter untuk membuat logika kontrol kompleks.
  6. Mensimulasikan program Timer/Counter dan membaca nilai ET (Elapsed Time) serta CV (Current Value).

2. Mengapa Timer dan Counter?

Dalam dunia nyata, hampir tidak ada proses industri yang hanya membutuhkan logika ON/OFF sederhana. Proses seperti pencampuran bahan kimia, pengisian cairan, pemanasan oven, atau pengepakan produk memerlukan waktu tertentu (Timer) dan penghitungan jumlah (Counter).

Timer dan Counter pada PLC menggantikan timer relay mekanis dan counter elektro-mekanis yang mahal, tidak akurat, dan sulit dikonfigurasi. Pada PLC modern (Siemens S7-1200), tersedia ratusan Timer dan Counter virtual sesuai kebutuhan.

3. Jenis-Jenis Timer (IEC Standard)

⏱ TON – Timer On Delay

Output (Q) ON setelah input (IN) terus ON selama durasi PT. Jika IN dimatikan sebelum PT tercapai, timer reset. Paling sering digunakan.

🔄 TOF – Timer Off Delay

Output (Q) langsung ON saat input ON. Ketika input OFF, output bertahan ON selama durasi PT sebelum akhirnya OFF. Contoh: kipas pendingin mesin tetap berputar 30 detik setelah mesin mati.

⚡ TP – Timer Pulse

Output menghasilkan pulsa durasi tetap (PT) setiap kali ada rising edge pada input. Selama PT belum selesai, input baru diabaikan. Contoh: membuat kedipan lampu durasi tetap.

4. Parameter Timer

Parameter Tipe Data Keterangan
IN BOOL Input enable timer. Rising edge memulai timer (TON/TP), falling edge memulai (TOF).
PT (Preset Time) TIME Durasi target timer. Format: T#5s (5 detik), T#500ms, T#2m30s. Resolusi: 1ms (S7-1200).
Q BOOL Output status timer. TRUE jika timer sudah mencapai PT (TON/TOF) atau sedang menghitung (TP).
ET (Elapsed Time) TIME Waktu yang telah berlalu sejak timer mulai (0 hingga PT). Berguna untuk monitoring progress.

5. Contoh Program Timer TON – Lampu Otomatis

Skenario: Tekan tombol (I0.0), lampu (Q0.0) menyala 5 detik kemudian secara otomatis.

Network 1 – Timer TON: delay nyala 5 detik |--[ I0.0 BTN ]--|TON|--( Q0.0 LAMP )--| IN Q PT: T#5s ET → MW0 Cara kerja: 1. Saat I0.0 = ON → Timer mulai menghitung (ET bertambah dari T#0s) 2. Setelah ET = PT (5 detik) → Q timer = TRUE → Q0.0 = ON 3. Saat I0.0 = OFF → timer reset, ET = 0, Q = FALSE → Q0.0 = OFF

6. Jenis-Jenis Counter (IEC Standard)

Jenis Singkatan Fungsi Contoh Aplikasi
Count Up CTU Menghitung naik dari 0 hingga PV. Q = TRUE saat CV ≥ PV. Reset dengan R. Menghitung botol yang lewat konveyor. Alarm saat 100 botol terpenuhi.
Count Down CTD Menghitung turun dari PV ke 0. Q = TRUE saat CV ≤ 0. Load ulang dengan LD. Sisa slot parkir. Load 50, turun tiap mobil masuk, alarm saat penuh (=0).
Count Up/Down CTUD Bisa naik (CU) dan turun (CD). Punya dua output: QU (≥ PV) dan QD (≤ 0). Manajemen stok gudang: barang masuk (CU) & keluar (CD).

7. Parameter Counter

Parameter Tipe Data Keterangan
CU / CD BOOL Input pulsa naik/turun. Counter menghitung setiap rising edge pada pin ini.
R / LD BOOL R = Reset CV ke 0 (CTU). LD = Load PV ke CV (CTD, digunakan untuk set ulang).
PV (Preset Value) INT/DINT Nilai target counter. Contoh: PV = 100 untuk menghitung 100 produk.
Q / QU / QD BOOL Output status. TRUE saat CV mencapai batas atas (CTU) atau batas bawah (CTD).
CV (Current Value) INT/DINT Nilai hitungan saat ini. Berguna untuk ditampilkan di HMI atau logika lanjutan.

8. Contoh Kombinasi Timer + Counter – Sistem Pengepakan

Skenario: Sensor (I0.0) menghitung produk di konveyor. Setiap 10 produk (PV=10), mesin bundling aktif selama 3 detik, lalu Counter direset otomatis.

Network 1 – Counter menghitung produk (CTU): |--[P I0.0 SENSOR]--|CTU|--( M0.1 FULL )--| CU Q R ← M0.2 PV: 10 CV → MW10 Network 2 – Timer aktif 3s saat counter penuh: |--[ M0.1 FULL ]--|TON|--( Q0.0 BUNDLER )--| IN Q PT: T#3s Network 3 – Reset counter setelah 3 detik bundling selesai: |--[ Q0.0 BUNDLER ]--( M0.2 RESET_CTR )--| Hasil: Setiap 10 produk → Bundler ON 3 detik → Counter reset → Hitung ulang
💡 Tips Penggunaan Timer di Siemens TIA Portal Di S7-1200, Timer IEC menggunakan Instance Data Block (IDB) tersendiri. Setiap instance Timer memiliki blok data unik (mis: "MyTimer" DB). Jangan menggunakan satu Instance DB untuk beberapa Timer — ini akan menyebabkan konflik nilai ET.
