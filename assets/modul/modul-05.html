Logika Interlock & Keselamatan

Prinsip interlock untuk mencegah konflik output, implementasi Emergency Stop, dan standar keselamatan industri ISO 13849.

Kelas: XII TEI & TPM
Durasi: 2 × 45 menit
Level: Menengah
Topik: 7 Sub-Topik
🔴 MODUL KESELAMATAN KRITIS Materi modul ini berkaitan langsung dengan keselamatan jiwa manusia dan mesin di lingkungan industri. Pahami setiap konsep dengan seksama. Kesalahan implementasi sistem E-Stop atau interlock dapat mengakibatkan cedera serius atau kematian.

1. Tujuan Pembelajaran

  1. Menjelaskan definisi dan fungsi interlock dalam sistem kontrol PLC.
  2. Membedakan software interlock dan hardware interlock beserta kelebihan masing-masing.
  3. Merancang program Ladder Diagram dengan interlock Forward/Reverse yang andal.
  4. Menjelaskan fungsi dan standar implementasi Emergency Stop (IEC 60204-1).
  5. Memahami konsep Safety Category (Cat.1–Cat.4) menurut ISO 13849.
  6. Mengimplementasikan program E-Stop dengan auto-restart prevention.
  7. Menganalisis kebenaran sistem interlock melalui tabel pengamatan logika.

2. Apa itu Interlock?

Interlock adalah mekanisme dalam program PLC (atau rangkaian hardware) yang secara sengaja mencegah dua atau lebih kondisi yang saling bertentangan (mutually exclusive) aktif secara bersamaan.

Contoh paling klasik adalah kontrol motor tiga fasa forward/reverse: jika kontaktor Forward (KM-FWD) dan kontaktor Reverse (KM-REV) aktif secara bersamaan, terjadi hubung singkat antar fasa yang dapat merusak motor dan menyebabkan kebakaran. Interlock mencegah hal ini terjadi.

3. Jenis-Jenis Interlock

Software Interlock

Implementasi dalam program Ladder: kontak NC dari output lawan dimasukkan ke dalam rung output yang sedang aktif. Murah dan fleksibel, namun bergantung pada kebenaran program PLC.

Hardware Interlock

Menggunakan kontak NC fisik dari kontaktor itu sendiri (auxiliary contact). Independen dari program PLC — lebih andal karena bekerja meski program crashed. WAJIB ada di aplikasi industri nyata.

Emergency Stop (E-Stop)

Tombol "jamur" merah yang memutus semua output berbahaya secara instan. Menggunakan kontak NC (fail-safe: kabel putus = sistem aman). Wajib sesuai IEC 60204-1.

Safety Relay Module

Modul elektronik khusus yang memonitor integritas sirkuit E-Stop secara terus-menerus. Diperlukan untuk aplikasi Safety Category 3 ke atas (PLd, PLe).

4. Prinsip Fail-Safe (Keamanan Default)

Desain sistem yang baik harus mengikuti prinsip Fail-Safe: sistem harus pergi ke kondisi AMAN secara otomatis jika terjadi kegagalan (putus kabel, short circuit, kehilangan daya).

Komponen Konfigurasi Fail-Safe Alasan
Tombol E-Stop Kontak NC (Normally Closed) Kabel putus = output mati (AMAN)
Tombol STOP Kontak NC (Normally Closed) Kegagalan = mesin berhenti (AMAN)
Tombol START Kontak NO (Normally Open) Kegagalan = mesin tidak start sendiri (AMAN)
Limit Switch Proteksi Kontak NC Kabel putus = gerakan terhenti (AMAN)
Output Aktuator Berbahaya Relay NC ke Safety Relay Kehilangan daya = aktuator berhenti (AMAN)

5. Program Interlock Forward/Reverse Motor

Network 1 – Motor Forward dengan software interlock: |--[ I0.0 BTN_FWD ]--+--[/ I0.1 STOP ]--[/ I0.3 ESTOP ]--[/ Q0.1 KM_REV ]--( Q0.0 KM_FWD )--| |--[ Q0.0 HOLD ]--+ ↑ self-hold ↑ interlock: NC kontak dari KM_REV Network 2 – Motor Reverse dengan software interlock: |--[ I0.2 BTN_REV ]--+--[/ I0.1 STOP ]--[/ I0.3 ESTOP ]--[/ Q0.0 KM_FWD ]--( Q0.1 KM_REV )--| |--[ Q0.1 HOLD ]--+ ↑ self-hold ↑ interlock: NC kontak dari KM_FWD Network 3 – Alarm E-Stop aktif: |--[/ I0.3 ESTOP ]--( Q0.2 ALARM_LAMP )--| Logika Kebenaran: - BTN_FWD + E-Stop normal → KM_FWD ON, KM_REV OFF ✓ - BTN_REV aktif saat FWD ON → KM_REV TETAP OFF (interlock) ✓ - E-Stop ditekan → SEMUA output mati, ALARM ON ✓

6. Safety Category Menurut ISO 13849

Kategori Tingkat Keandalan Konfigurasi Contoh Aplikasi
Cat. B Sangat Rendah Satu channel, tanpa diagnostik Lampu, buzzer non-kritis
Cat. 1 Rendah Satu channel, menggunakan komponen proven Mesin ringan, kecepatan rendah
Cat. 2 Sedang Satu channel + fungsi tes periodik Mesin semi-otomatis
Cat. 3 Tinggi Dua channel redundant, diagnostik single fault Robot, press, mesin berat
Cat. 4 Sangat Tinggi (PLe) Dua channel + cross-monitoring penuh Robotik, mesin berbahaya tinggi

7. Anti Auto-Restart Setelah E-Stop

Ini adalah persyaratan WAJIB menurut standar keselamatan: setelah E-Stop dilepas (reset), mesin TIDAK BOLEH langsung beroperasi sendiri. Operator harus menekan tombol START secara sadar setelah memverifikasi area aman.

Implementasi menggunakan SR Latch: Network – SR Latch untuk kondisi E-Stop: |--[/ I0.3 ESTOP ]--( S: M0.0 ESTOP_FLAG )--| → SET saat E-Stop aktif |--[ I0.0 START ]--( R: M0.0 ESTOP_FLAG )--| → RESET hanya saat START ditekan ulang Network – Output diblokir saat ESTOP_FLAG aktif: |--[ M0.0 ESTOP_FLAG / NC ]--( ... output kontrol motor ... )--| Hasilnya: Mesin hanya bisa jalan kembali setelah operator: 1. Memastikan area aman 2. Melepas (memutar) tombol E-Stop 3. Menekan tombol START secara eksplisit

  • ISO 13849-1: Bagian-bagian yang terkait keselamatan dari sistem kontrol.
