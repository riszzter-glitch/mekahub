JS-04: Sistem Interlock & Emergency Stop

Modul 05 – Logika Interlock & Keselamatan | MekaHub PLC Learning Platform

Kelas: XII TEI/TPM
Durasi: 2 × 45 menit
Level: Menengah
Versi: 1.0 (2025/2026)
🔴 PERINGATAN KESELAMATAN UTAMA Jobsheet ini membahas sistem Emergency Stop (E-Stop) yang merupakan komponen keselamatan kritis. Dalam kondisi nyata, E-Stop WAJIB dikategori keselamatan (Safety Category per ISO 13849). Jangan memodifikasi atau men-bypass sistem E-Stop di lingkungan industri nyata tanpa otorisasi!

A. Tujuan Pembelajaran

  1. Memahami konsep interlock dalam sistem kontrol PLC untuk mencegah konflik aktuator.
  2. Merancang dan memprogram sistem Emergency Stop (E-Stop) yang andal pada PLC S7-1200.
  3. Mengimplementasikan Safety Relay dan kategori keselamatan (Safety Category 1–4).
  4. Menguji fungsi E-Stop secara sistematis dan mendokumentasikan hasilnya.

B. Konsep Interlock

Interlock adalah mekanisme program yang mencegah dua atau lebih aktuator yang saling bertentangan aktif secara bersamaan. Contoh klasik adalah kontaktor Forward dan Reverse pada motor – jika keduanya aktif sekaligus, terjadi hubung singkat fasa.

Software Interlock

Implementasi melalui program Ladder (kontak NC dari output lawan dimasukkan dalam rung). Bisa gagal jika program bermasalah.

Hardware Interlock

Menggunakan kontak NC fisik dari kontaktor itu sendiri. Lebih andal karena independen dari program PLC.

Emergency Stop (E-Stop)

Tombol jamur merah NC yang memutus semua aktuator berbahaya secara instan. Wajib ada pada setiap mesin industri (IEC 60204-1).

Safety Category

Tingkatan keandalan sistem keselamatan: Cat.1 (sederhana) hingga Cat.4 (redundant, diagnostik penuh, PLe).

C. Peta I/O (Address PLC)

Tag Address Tipe Fungsi
BTN_FWD I0.0 Input DI Tombol Forward motor (NO)
BTN_REV I0.1 Input DI Tombol Reverse motor (NO)
BTN_STOP I0.2 Input DI Tombol Stop normal (NC)
E_STOP I0.3 Input DI Tombol Emergency Stop (NC – mustn't bypass!)
SENSOR_LIMIT_L I0.4 Input DI Limit switch kiri (mekanik interlock posisi)
SENSOR_LIMIT_R I0.5 Input DI Limit switch kanan (mekanik interlock posisi)
KM_FWD Q0.0 Output DO Kontaktor Forward
KM_REV Q0.1 Output DO Kontaktor Reverse
LAMP_RUN Q0.2 Output DO Lampu hijau – mesin berjalan
LAMP_ESTOP Q0.3 Output DO Lampu merah berkedip – E-Stop aktif
BUZZER Q0.4 Output DO Buzzer alarm saat E-Stop

D. Logika Safety: Aturan Utama

🔴 Aturan Tak Dapat Dikompromikan E-Stop (I0.3) harus ditempatkan di SETIAP rung yang mengontrol aktuator berbahaya. Jika I0.3 = FALSE (trip), SEMUA output aktuator HARUS langsung mati, tanpa pengecualian.

E. Program Ladder Diagram

Network 1 – Kontrol Forward dengan interlock penuh:

|--[I0.0 BTN_FWD]--+--[I0.2 STOP /NC]--[I0.3 ESTOP /NC]--[/Q0.1 KM_REV interlock]--[/I0.5 LimSwR]--| (Q0.0 KM_FWD) | [Q0.0 self-hold] |

Network 2 – Kontrol Reverse dengan interlock penuh:

|--[I0.1 BTN_REV]--+--[I0.2 STOP /NC]--[I0.3 ESTOP /NC]--[/Q0.0 KM_FWD interlock]--[/I0.4 LimSwL]--| (Q0.1 KM_REV) | [Q0.1 self-hold] |

Network 3 – Lampu RUN (aktif saat mesin bergerak):

|--[Q0.0]--+--[Q0.1]--| (Q0.2 LAMP_RUN)

Network 4 – Alarm E-Stop (Lampu merah berkedip + Buzzer):

|--[/I0.3 ESTOP]--| (Q0.3 LAMP_ESTOP + Q0.4 BUZZER)
⚠️ Perhatian Reset E-Stop Setelah E-Stop dilepas (reset), sistem TIDAK boleh langsung kembali beroperasi sendiri (auto-restart). Operator harus menekan tombol START ulang setelah memastikan area aman. Implementasikan flag SR latch untuk kondisi ini.

F. Langkah Kerja

  1. Buat project TIA Portal, konfigurasi I/O sesuai tabel bagian C.
  2. Masukkan program Ladder pada OB1 (Network 1–4).
  3. Simulasikan di PLC-SIM – perhatikan bahwa saat I0.3 dimatikan (E-Stop aktif), semua output Q langsung mati.
  4. Uji interlock: aktifkan Q0.0 (FWD), lalu coba aktifkan Q0.1 (REV) → harus TIDAK bisa aktif bersamaan.
  5. Uji Limit Switch: jalankan motor FWD → saat SENSOR_LIMIT_R aktif, motor harus berhenti otomatis.
  6. Uji E-Stop: saat motor berjalan, aktifkan E-Stop → motor berhenti, LAMP_ESTOP berkedip, BUZZER berbunyi.
  7. Catat semua hasil pada tabel pengamatan.
  8. Lepas E-Stop → amati: motor TIDAK boleh start otomatis.

G. Tabel Pengamatan

No Kondisi Uji KM_FWD KM_REV LAMP_ESTOP BUZZER ✓/✗
1 Tekan BTN_FWD, E-Stop normal ON OFF OFF OFF
2 FWD aktif + Tekan BTN_REV ON OFF OFF OFF
3 FWD aktif + Aktifkan E-Stop OFF OFF ON (kedip) ON
4 E-Stop dilepas, tanpa tekan START OFF OFF OFF OFF
5 SENSOR_LIMIT_R aktif saat FWD OFF OFF OFF OFF
6 Tekan STOP saat motor berjalan OFF OFF OFF OFF

H. Pertanyaan Evaluasi

  1. Jelaskan perbedaan antara software interlock dan hardware interlock, dan mengapa keduanya sebaiknya diterapkan bersamaan!
  2. Mengapa tombol Emergency Stop menggunakan kontak NC (Normally Closed) bukan NO? Apa akibatnya jika kabel E-Stop putus?
  3. Apa yang dimaksud dengan "Safety Category" menurut ISO 13849? Jelaskan perbedaan antara Category 1 dan Category 3!
  4. Mengapa auto-restart setelah E-Stop dilepas dianggap berbahaya? Bagaimana cara mencegahnya dalam program PLC?
  5. Dalam sebuah lini produksi dengan 5 mesin, bagaimana cara mengimplementasikan E-Stop yang menghentikan semua mesin secara bersamaan?
