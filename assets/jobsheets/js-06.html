🏆 Tentang Proyek Akhir Ini Jobsheet ini adalah puncak dari seluruh pembelajaran PLC di MekaHub. Peserta didik akan mengintegrasikan semua kompetensi yang telah dipelajari — dari wiring input/output, logika kontrol, timer/counter, interlock, hingga HMI — untuk membangun sistem Smart Factory Mini yang fungsional secara penuh.

A. Tujuan Pembelajaran

  1. Mengintegrasikan seluruh kompetensi PLC yang telah dipelajari dalam satu proyek terpadu.
  2. Merancang sistem kontrol otomatis untuk lini produksi mini (konveyor + sortir + penghitung).
  3. Membuat antarmuka HMI sederhana untuk monitoring dan kontrol operator.
  4. Menerapkan sistem keselamatan lengkap (E-Stop, interlock, alarm) pada sistem produksi.
  5. Mendokumentasikan proyek secara profesional seperti layaknya engineer industri.

B. Deskripsi Sistem Smart Factory Mini

Smart Factory Mini adalah simulasi lini produksi yang terdiri dari tiga subsistem utama:

01

Konveyor Utama

Motor konveyor membawa produk dari stasiun loading ke stasiun sorting. Kecepatan diatur, terdapat sensor deteksi produk.

02

Sistem Sortir

Sensor kapasitif/induktif membedakan jenis produk (logam/non-logam). Pneumatik pusher memisahkan produk ke lajur berbeda.

03

HMI & Monitoring

Panel HMI menampilkan status sistem, counter produk per kategori, alarm aktif, dan tombol kontrol operator.

E. Logika Program Utama (Sequence)

Program dibagi menjadi beberapa Function Block (FB) terstruktur:

  1. FB_Safety – Mengelola E-Stop, alarm, dan kondisi keselamatan. Dieksekusi pertama di OB1.
  2. FB_Conveyor – Kontrol start/stop konveyor, self-hold, interlock limit switch.
  3. FB_Sorting – Logika sortir: deteksi produk → delay timer → aktifkan pusher yang sesuai.
  4. FB_Counter – Menghitung produk logam (CTU1) dan non-logam (CTU2), reset via HMI.
  5. FB_HMI – Menulis/membaca tag HMI: status, counter, alarm.
[OB1 – Main Cycle] CALL FB_Safety (DB_Safety) // Prioritas tertinggi CALL FB_Conveyor (DB_Conveyor) // Hanya jalan jika Safety OK CALL FB_Sorting (DB_Sorting) // Hanya jalan saat konveyor ON CALL FB_Counter (DB_Counter) // Update counter produk CALL FB_HMI (DB_HMI) // Sinkronisasi data HMI

F. Tahapan Proyek (Milestone)

1

Pertemuan 1–2: Perencanaan & Desain (180 menit)

Buat diagram blok sistem, tentukan spesifikasi I/O, gambar wiring diagram, buat jadwal pengerjaan kelompok, dan bagi tugas anggota.

2

Pertemuan 3: Pemrograman PLC (90 menit)

Buat project TIA Portal, konfigurasi hardware, buat semua FB (FB_Safety, FB_Conveyor, FB_Sorting, FB_Counter), uji logika dengan PLC-SIM.

3

Pertemuan 4: Wiring & Hardware (90 menit)

Rangkai semua komponen pada trainer sesuai diagram wiring. Verifikasi koneksi sebelum power-on. Upload program ke PLC.

4

Pertemuan 5: Konfigurasi HMI (90 menit)

Buat layar HMI: Main Screen (status + counter), Alarm Screen, Control Screen. Hubungkan tag HMI dengan tag PLC. Test komunikasi Profinet.

5

Pertemuan 6: Pengujian & Presentasi (90 menit)

Uji semua skenario operasi, catat hasil di tabel pengamatan, buat laporan, dan presentasikan proyek di depan penguji.

G. Skenario Pengujian

No Skenario Uji Hasil yang Diharapkan Hasil ✓/✗
1 Tekan START → sistem ready Motor konveyor ON, LAMP_RUN ON
2 Produk logam terdeteksi sensor Delay 1s → PUSHER_LOGAM aktif 0.5s
3 Produk non-logam terdeteksi Delay 1s → PUSHER_NONLOGAM aktif 0.5s
4 Counter logam setelah 5 produk HMI menampilkan counter = 5
5 Aktifkan E-Stop saat sistem ON Semua output OFF, LAMP_ALARM ON, BUZZER ON
6 Lepas E-Stop → tanpa tekan START Sistem TIDAK auto-restart
7 Reset counter via tombol HMI Kedua counter kembali 0 di HMI
8 Limit Switch kanan aktif (produk macet) Konveyor berhenti, alarm muncul di HMI
